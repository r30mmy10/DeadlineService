# DeadlineService

> Веб-приложение для контроля задач и дедлайнов с пользовательской и административной частью.

---

## О проекте

**DeadlineService** — это учебный командный проект, посвящённый автоматизации работы с задачами и дедлайнами.  
Система позволяет пользователю регистрироваться, входить в аккаунт, управлять своими задачами и отслеживать сроки выполнения.  
Для администратора предусмотрена отдельная панель с доступом ко всем пользователям и задачам системы.

Проект реализуется как распределённая система:

- **C# / ASP.NET Core** — основной backend, аутентификация, работа с задачами, пользователями и БД
- **Go** — сервис уведомлений о приближающихся дедлайнах
- **PostgreSQL** — система хранения данных
- **HTML / CSS / JavaScript** — минимальный пользовательский интерфейс

---

## Основной функционал

### Пользовательская часть
- регистрация
- вход в систему
- просмотр своих задач
- создание новой задачи
- редактирование своей задачи
- удаление своей задачи

### Административная часть
- просмотр всех пользователей
- просмотр всех задач
- редактирование любой задачи от имени администратора

### Планируемое развитие
- email-уведомления о приближающихся дедлайнах
- внутренние уведомления в системе
- история уведомлений
- повторные попытки отправки уведомлений
- блокировка пользователей

## Архитектура проекта

Проект разделён на несколько логических частей.

### 1. Основной backend на C#
Реализован на **ASP.NET Core Web API** и отвечает за:
- регистрацию и авторизацию пользователей
- выдачу JWT-токенов
- CRUD-операции над задачами
- административные endpoint’ы
- взаимодействие с PostgreSQL через Entity Framework Core

### 2. Сервис уведомлений на Go
Предназначен для:
- периодической проверки дедлайнов
- генерации уведомлений
- отправки email
- хранения истории попыток доставки

### 3. Клиентская часть
Простой web-интерфейс, состоящий из:
- страницы входа и регистрации
- страницы задач
- административной страницы

### 4. База данных PostgreSQL
Используется для хранения:
- пользователей
- задач
- настроек уведомлений
- уведомлений
- истории доставки уведомлений

## База данных

В проекте используется **PostgreSQL**.  
На текущем этапе база подключена и миграции применяются через **Entity Framework Core**.

Основные сущности:
- `Users`
- `Tasks`
- `NotificationSettings`

В дальнейшей версии планируется использование:
- `Notifications`
- `NotificationDeliveryHistory`

## Как запустить проект

### 1. Клонировать репозиторий
```bash
git clone https://github.com/r30mmy10/DeadlineService.git
cd DeadlineService
```

### 2. Убедиться, что установлен PostgreSQL
Создать базу данных:

```bash
createdb deadline_service_db
```

### 3. Настроить строку подключения
В файле `appsettings.json` указать актуальные параметры подключения к PostgreSQL.

Пример:

```json
"ConnectionStrings": {
  "DefaultConnection": "Host=localhost;Port=5432;Database=deadline_service_db;Username=heragromov"
}
```

### 4. Применить миграции
```bash
dotnet ef database update --project DeadlineService.Api
```

### 5. Запустить проект
```bash
dotnet run --project DeadlineService.Api
```

## Роли пользователей

В системе предусмотрены две роли:

### `User`
Обычный пользователь, который может:
- входить в систему
- управлять только своими задачами

### `Admin`
Администратор, который может:

- просматривать все задачи
- редактировать любую задачу

## Git workflow

Для командной разработки используется один общий репозиторий и отдельные ветки:

- `main` — стабильная ветка
- `feature/csharp-app` — основная ветка для C#-части и интерфейса
- `feature/go-notifications` — ветка для Go-сервиса уведомлений

Разработка ведётся в feature-ветках с последующим слиянием в `main`.
